# Python赋值

**Python 里“赋值”默认是绑定对象（引用），不是拷贝。**  
真正拷贝要显式做（浅拷贝/深拷贝）。

---

## 1) 赋值：引用（不是拷贝）

```python
a = [1, 2]
b = a
b.append(3)
print(a)  # [1, 2, 3]
```

`b = a` 后，`a` 和 `b` 指向同一个列表对象。

---

## 2) 可变对象 vs 不可变对象（很关键）

### 可变对象（list/dict/set）
改其中一个，另一个通常也变（因为同对象）：

```python
x = {"n": 1}
y = x
y["n"] = 99
print(x["n"])  # 99
```

### 不可变对象（int/str/tuple）
看起来像“改了”，其实是新建对象再绑定：

```python
a = 10
b = a
b = b + 1
print(a, b)  # 10 11
```

这里不是把 `10` 改成 `11`，而是 `b` 重新指向了新对象 `11`。

---

## 3) 切片/构造器：常见“浅拷贝”

```python
a = [1, 2, 3]
b = a[:]          # 或 list(a), a.copy() 表示这个列表的整段切片（从头到尾）。
b.append(4)
print(a)  # [1, 2, 3]
print(b)  # [1, 2, 3, 4]
```

这时外层列表已分开。

---

## 4) 浅拷贝陷阱（嵌套结构）

```python
a = [[1], [2]]
b = a[:]          # 浅拷贝：只拷贝外层
b[0].append(9)
print(a)  # [[1, 9], [2]]
```

因为内层小列表还是同一个对象。

---

## 5) 深拷贝（嵌套也独立）

```python
import copy

a = [[1], [2]]
b = copy.deepcopy(a)
b[0].append(9)
print(a)  # [[1], [2]]
print(b)  # [[1, 9], [2]]
```

---

## 6) 函数传参本质：传对象引用（对象共享）

```python
def f(lst):
    lst.append(100)

x = [1, 2]
f(x)
print(x)  # [1, 2, 100]
```

函数里改的是同一个列表对象。

---

# 对比

```
a = [1, 2, 3]
b = a            # b 和 a 指向同一个列表
a = [9]          # 重新绑定 a
print(b)         # [1, 2, 3]  (b 不受影响)
a = b            # 让 a 再次指回同一个对象
a[:] = [7, 8]    # 原地改内容
print(b)         # [7, 8]      (b 跟着变)
```

这里最容易混淆的是“改对象”和“变量重新绑定”。

这个例子其实是对的，关键点是：

- `b = a` 后，二者确实引用同一个列表对象（比如对象A：`[1,2,3]`）
- `a = [9]` 不是修改对象A，而是让变量 `a` 指向了一个新对象B：`[9]`
- `b` 仍然指向旧对象A，所以 `print(b)` 还是 `[1,2,3]`

# 结论

**一句话记忆：**

- **`a[0] = ...`、`a.append(...)`、`a[:] = ...`：改对象（共享引用会一起看到变化）**
- **`a = ...`：改变量指向（别名变量不受影响）**
